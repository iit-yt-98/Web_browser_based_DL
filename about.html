Explanation of what's happening!

We are using transfer learning in this project by using the MobileNet model from TensorflowJS library and this model is choosen as the heavier the model gets the more the time it takes to load the model and also the performance is also a big factor which favors small yet okayish models like MobileNet.
In JavaScript file we are taking a async function that waits for a promise using the await keyword.These special keywords are important and if you are not aware of these please watch the JS video by Akshay Saini on Youtube.
So basically we are loading the model and this model is then fed some images which are captured real time using the webcam of the user. Now to convert this image to a tensor we'll have to use a data API from TensorFlowJS and now this tensor can be fed to the model which will inturn classify it as a particular object.
Now you must be thinking that I have not preprocessed the image well actually this data API that we are using it handles all the things on it's own and that is a disadvantage of this project because you'll see that not always it performs absolutely the way it is desired.

Improvements

I would love to include some preprocessing on the captured image before it's fed to the model.
The model although is light needs to perform better as we have cases where it fails to identify and gives very less probability.
The image captured by the webcams are mostly very noisy so a noise removing filter is important to improve results.
